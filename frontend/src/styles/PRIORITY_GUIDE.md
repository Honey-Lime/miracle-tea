# Шпаргалка по приоритетам стилей

## Порядок приоритетов (от низкого к высокому)

```
┌─────────────────────────────────────────────────────────────┐
│  НИЗКИЙ ПРИОРИТЕТ (переопределяется)                        │
│  ┌───────────────────────────────────────────────────────┐  │
│  │ 1. index.css + components.css                         │  │
│  │    - Базовые стили, reset, утилиты                    │  │
│  │    - Общие компоненты (кнопки, теги, формы)           │  │
│  └───────────────────────────────────────────────────────┘  │
│                          ↓                                  │
│  ┌───────────────────────────────────────────────────────┐  │

│  │ 3. pages.css (ВЫСОКИЙ ПРИОРИТЕТ)                      │  │

```

## Быстрый справочник

### Куда добавить стиль?

| Тип стиля                    | Куда добавить                                                    |
| ---------------------------- | ---------------------------------------------------------------- |
| Кнопки, теги, формы, модалки | `components.css`                                                 |
| Шапка, подвал, калькуляторы  | Создать файл в `components/` + импорт в `components-imports.css` |
| Стили конкретной страницы    | Создать файл в `pages/` + импорт в `pages.css`                   |
| Переопределение для страницы | Файл страницы (например, `CatalogPage.css`)                      |

### Примеры

#### Добавить общую кнопку

```css
/* components.css */
.btn-new {
  background: #ff9800;
  color: white;
}
```

#### Переопределить для страницы

```css
/* CatalogPage.css */
.catalog-page .btn-new {
  background: #4a6fa5; /* Другой цвет на странице каталога */
}
```

## Отладка

### Стили не применяются?

1. **Проверьте порядок в `main.jsx`**:

   ```javascript
   import "./styles/index.css"; // 1
   import "./styles/components-imports.css"; // 2
   import "./styles/pages.css"; // 3
   ```

2. **Проверьте импорт в соответствующем файле**:

   - Для компонентов: `components-imports.css`
   - Для страниц: `pages.css`

3. **Проверьте специфичность в DevTools**

### Конфликт стилей?

Используйте более специфичный селектор:

```css
/* ✗ Плохо - может переопределиться */
.btn {
  ...;
}

/* ✓ Хорошо - специфичный селектор */
.product-page .product-info .btn {
  ...;
}
```

## Чеклист при добавлении нового стиля

- [ ] Это общий стиль (кнопки, формы)? → `components.css`
- [ ] Это стиль компонента (Header, Footer)? → `components/*.css` + `components-imports.css`
- [ ] Это стиль страницы? → `pages/*.css` + `pages.css`
- [ ] Нужно переопределить общий стиль? → Используйте специфичный селектор в файле страницы
- [ ] Удалён ли импорт CSS из JSX-файла? → Должен быть только централизованный импорт
