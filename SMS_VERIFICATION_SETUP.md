# Настройка SMS-верификации

## 1. Регистрация в SMS.ru

1. Перейдите на https://sms.ru/
2. Зарегистрируйте аккаунт
3. Получите API ключ в личном кабинете (раздел "Настройки" → "API")

## 2. Настройка бэкенда

### Обновите `.env` файл

Откройте `backend/.env` и замените:

```env
SMS_API_KEY=your_sms_api_key
```

на ваш реальный API ключ:

```env
SMS_API_KEY=ваш_api_ключ_с_sms.ru
SMS_SENDER_NAME=MiracleTea
```

### Примечание о имени отправителя

- По умолчанию установлено `MiracleTea`
- Для использования своего имени отправителя его нужно согласовать с SMS.ru
- В тестовом режиме можно использовать стандартное имя

## 3. Тестирование

### Режим разработки

В режиме разработки (`NODE_ENV=development`) код подтверждения возвращается в ответе API и отображается в интерфейсе. Это удобно для тестирования без реальных SMS.

### Боевой режим

В production режиме код **не возвращается** в ответе API — он отправляется только по SMS.

## 4. Как это работает

### Процесс регистрации:

1. Пользователь вводит номер телефона
2. Нажимает "Получить код"
3. Получает SMS с 6-значным кодом
4. Вводит код в поле
5. После подтверждения вводит имя и пароль
6. Регистрация завершена

### Защита от злоупотреблений:

- Код действителен 5 минут
- Повторная отправка кода возможна через 1 минуту
- Использованные коды помечаются и не могут быть использованы повторно
- Старые коды автоматически удаляются из базы данных

## 5. Модель данных

Коды хранятся в коллекции `SmsCode` со следующими полями:

- `phone` — номер телефона
- `code` — код подтверждения
- `expiresAt` — время истечения (автоматическое удаление)
- `isUsed` — флаг использования
- `createdAt` — время создания

## 6. API эндпоинты

### Отправка кода

```
POST /api/sms/send-code
Body: { "phone": "79991234567" }
```

### Проверка кода

```
POST /api/sms/verify-code
Body: { "phone": "79991234567", "code": "123456" }
```

## 7. Тарифы SMS.ru

На момент написания:

- Тестовый период: 10 SMS бесплатно
- По России: ~1.5-3 ₽ за SMS в зависимости от региона
- При больших объёмах действуют оптовые тарифы

Актуальные тарифы смотрите на https://sms.ru/

## 8. Безопасность

- API ключ хранится только на сервере в `.env`
- Коды автоматически удаляются после использования или истечения
- Один код можно использовать только один раз
- При множественных неудачных попытках можно добавить rate limiting
