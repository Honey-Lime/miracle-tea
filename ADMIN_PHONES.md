# Управление админскими email

## Настройка

Список админских email хранится в файле `backend/.env` в переменной `ADMIN_EMAILS`.

### Формат записи

```env
ADMIN_EMAILS=admin1@yandex.ru,admin2@yandex.ru,admin3@yandex.ru
```

- Email-адреса разделяются **запятой**
- Пробелы лучше не использовать
- Регистр не важен, адреса приводятся к нижнему регистру
- Количество email не ограничено

### Примеры

✅ Правильно:

```env
ADMIN_EMAILS=admin1@yandex.ru
ADMIN_EMAILS=admin1@yandex.ru,admin2@yandex.ru
ADMIN_EMAILS=admin1@yandex.ru,admin2@yandex.ru,admin3@yandex.ru
```

❌ Неправильно:

```env
ADMIN_EMAILS=
ADMIN_EMAILS=admin1
ADMIN_EMAILS=admin1@yandex.ru;admin2@yandex.ru
```

## Как это работает

### При регистрации нового пользователя

1. Система проверяет, есть ли email в списке `ADMIN_EMAILS`
2. Если есть — пользователь автоматически создаётся с правами `isAdmin: true`
3. Если нет — создаётся как обычный пользователь

### При входе существующего пользователя

1. Система проверяет, есть ли email в списке `ADMIN_EMAILS`
2. Если email добавлен в список, но у пользователя нет прав — права автоматически выдаются
3. Если email удалён из списка, но у пользователя есть права — права автоматически отзываются

### Динамическое обновление

Изменения в `.env` вступают в силу **после перезапуска сервера**.

## Добавление нового админа

1. Откройте `backend/.env`
2. Добавьте email в конец списка `ADMIN_EMAILS` через запятую
3. Сохраните файл
4. Перезапустите сервер

## Удаление админа

1. Откройте `backend/.env`
2. Удалите email из списка `ADMIN_EMAILS`
3. Сохраните файл
4. Перезапустите сервер

## Проверка текущих админов

### Через базу данных MongoDB

```javascript
db.users.find({ isAdmin: true }, { name: 1, email: 1, isAdmin: 1 });
```

## Безопасность

- Файл `.env` не должен попадать в Git
- Храните админские email только у доверенных лиц
- При компрометации email удалите его из списка и смените пароль пользователю
