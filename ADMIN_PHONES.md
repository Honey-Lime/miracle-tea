# Управление админскими номерами телефонов

## Настройка

Список админских номеров хранится в файле `backend/.env` в переменной `ADMIN_PHONES`.

### Формат записи

```env
ADMIN_PHONES=79202115108,79991234567,79009876543
```

- Номера указываются **без знака `+`**
- Без пробелов, скобок и дефисов
- Разделяются **запятой** без пробелов
- Максимальное количество номеров не ограничено

### Примеры

✅ Правильно:

```env
ADMIN_PHONES=79202115108
ADMIN_PHONES=79202115108,79991234567
ADMIN_PHONES=79202115108,79991234567,79009876543
```

❌ Неправильно:

```env
ADMIN_PHONES=+79202115108          # Есть знак +
ADMIN_PHONES=79202115108, 79991234567  # Есть пробел после запятой
ADMIN_PHONES=7 (920) 211-51-08     # Есть пробелы и скобки
```

## Как это работает

### При регистрации нового пользователя

1. Система проверяет, есть ли номер в списке `ADMIN_PHONES`
2. Если есть — пользователь автоматически создаётся с правами `isAdmin: true`
3. Если нет — создаётся как обычный пользователь

### При входе существующего пользователя

1. Система проверяет, есть ли номер в списке `ADMIN_PHONES`
2. Если номер добавлен в список, но у пользователя нет прав — права автоматически выдаются
3. Если номер удалён из списка, но у пользователя есть права — права автоматически отзываются

### Динамическое обновление

Изменения в `.env` вступают в силу **после перезапуска сервера**.

После перезапуска:

- При следующем входе пользователя его права будут обновлены автоматически
- Не требуется ручное изменение пользователей в базе данных

## Добавление нового админа

1. Откройте `backend/.env`
2. Добавьте номер в конец списка `ADMIN_PHONES` через запятую
3. Сохраните файл
4. Перезапустите сервер (остановите и запустите заново)

**Пример:**

Было:

```env
ADMIN_PHONES=79202115108
```

Стало:

```env
ADMIN_PHONES=79202115108,79991234567
```

## Удаление админа

1. Откройте `backend/.env`
2. Удалите номер из списка `ADMIN_PHONES`
3. Сохраните файл
4. Перезапустите сервер

**Пример:**

Было:

```env
ADMIN_PHONES=79202115108,79991234567
```

Стало:

```env
ADMIN_PHONES=79202115108
```

## Проверка текущих админов

### Через базу данных MongoDB

Подключитесь к MongoDB и выполните запрос:

```javascript
db.users.find({ isAdmin: true }, { name: 1, phone: 1, isAdmin: 1 });
```

### Через админскую панель

Войдите под учётной записью администратора и перейдите в раздел управления пользователями (если реализовано).

## Безопасность

- Файл `.env` **не должен** быть закоммичен в Git (добавлен в `.gitignore`)
- Храните файл `.env` в безопасности на сервере
- При компрометации номера — удалите его из списка и смените пароль пользователю
- Регулярно проверяйте список админских номеров на актуальность
