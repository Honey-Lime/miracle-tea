# Miracle Tea – магазин чая

Полнофункциональный веб-сайт для продажи чая с React-фронтендом и Node.js-бэкендом.

## Функциональность

- **Главная страница** – заголовок, преимущества, калькулятор доставки СДЭК, контакты.
- **Каталог товаров** – список всех продуктов из базы данных.
- **Страница товара** – детальное описание, выбор количества грамм, добавление в корзину.
- **Корзина** – управление количеством, удаление, проверка минимального заказа (50 г).
- **Оформление заказа** – выбор доставки (самовывоз/СДЭК), расчёт стоимости.
- **Личный кабинет** – регистрация/вход по email, история заказов.
- **Административная часть** – управление товарами и заказами через API.

## Технологии

- **Фронтенд**: React 18, Vite, React Router, Context API, Axios, CSS.
- **Бэкенд**: Node.js, Express, MongoDB, Mongoose, JWT, bcrypt.
- **База данных**: MongoDB.

## Быстрый старт

### Предварительные требования

- Node.js 18+
- MongoDB (опционально, можно использовать облачную базу)
- npm или yarn

### Установка

1. Клонируйте репозиторий:

   ```bash
   git clone <repo-url>
   cd miracle-tea
   ```

2. Установите зависимости бэкенда:

   ```bash
   cd backend
   npm install
   ```

3. Установите зависимости фронтенда:

   ```bash
   cd ../frontend
   npm install
   ```

4. Настройте переменные окружения:
   - В файле `backend/.env` укажите:
     ```
     PORT=5000
     MONGODB_URI=mongodb://localhost:27017/miracle_tea
     JWT_SECRET=your_secret_key
     ```

### Запуск вручную

1. Запустите MongoDB (если используете локально):

   ```bash
   mongod --dbpath ./database
   ```

2. Запустите бэкенд (в отдельном терминале):

   ```bash
   cd backend
   npm run dev
   ```

3. Запустите фронтенд (в другом терминале):

   ```bash
   cd frontend
   npm run dev
   ```

4. Откройте браузер по адресу [http://localhost:3000](http://localhost:3000).

### Автоматический запуск (Windows)

Используйте `run.bat` из корневой директории:

```
run.bat
```

Скрипт проверит наличие MongoDB в PATH, и если она найдена, запустит сервер MongoDB с базой данных в папке `./database` (создаст её при необходимости). Затем запустит бэкенд и фронтенд в отдельных окнах. После нажатия любой клавиши в окне запуска все процессы (MongoDB, бэкенд, фронтенд) будут остановлены.

**Примечание:** Убедитесь, что MongoDB установлена и добавлена в переменную окружения PATH, иначе скрипт выведет предупреждение и продолжит работу (предполагается, что MongoDB уже запущена вручную или используется облачная база).

## API Endpoints

- `GET /api/products` – список товаров
- `POST /api/auth/login` – вход/регистрация
- `POST /api/orders` – создание заказа
- Полный список смотрите в коде бэкенда.

## Деплой

Для продакшн-деплоя рекомендуется:

- Использовать облачную MongoDB (Atlas)
- Настроить переменные окружения на хостинге
- Собрать фронтенд (`npm run build`) и раздать статику через Nginx
- Запустить бэкенд с помощью PM2
