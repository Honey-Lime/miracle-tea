Магазин чая

Создай простой сайт-магазин на архитектуре react, в связке с mongodb.

Сайт должен состоять из страниц:
    Главная страница
        Первый экран
            Заголовок "Чудо чай"
            Второстепенный заголовок "Здесь вы можете заказать качественный чай по лояльной цене"

        Блок преимущества
            Вмещает в себя 3 блока с картинкой и текстов в горизонтальном расположении
            1 "Минимальный заказ 50г"
            2 "Цена не завышена, ведь доставка не включена в стоимость"
            3 "Пробники по 10г, от 5 сортов чая"
        
        Блок доставка
            Текст: "Мы находимся в воронеже и осуществляем доставку через СДЕК. Ниже вы можете узнать стоимость доставки. Или посмотреть пункты самовывоза(в данный момент доступен только Воронеж."
            Блок калькулятора доставки.
                Нужно вставить калькулятор доставки СДЕК.
        
        Блок о нас:
            По всем вопросам вы можете обратится в чат на сайте или по электронной почте vikdub01@mail.ru

    Страница каталога
        В странице каталога должен список всех товаров
    Страница товара
        Страница товара состоит из окошка с просмотром фото и видео, ниже находится описание чая.
        Справа отображается цена и есть кнопка выбора кол-ва грамм для добавления в корзину, а также кнопка Хочу пробник.
        Минимальное кол-во грамм для заказа - 50г, порог увеличения кол-ва грамм - 2 кнопки +10г +50г. Заказ не может быть больше, чем осталось его на складе.
        На кнопке добавить в корзину отображается общее колво грамм и общая цена.
        При добавлении в корзину выбор кол-ва грамм обнуляется.
        Для каждого наполнения корзины есть ограничение - 1 пробник каждого вида чая. Пробники стоят также как и обычный чай

        В хидере есть список всех страниц и кнопка Корзина. При переходе в корзину мы видим список всех заказов. Для каждого из заказов можно изменить кол-во грамм и удалить из списка вовсе.
        Внизу корзины отображается общая стомость заказа.
        Также есть кнопка оформить заказ. При нажатии пользователя отправляет на страницу оформления заказа. На этой странице есть список со всеми пунктами которые он выбрал, но без возможности изменения или удаления пункта. Под списком отображается общая стоимость заказа.

        Ниже списка заказа стоит блок настройки доставки на выбор есть либо самовывоз(доставка бесплатно). Либо доставка СДЕК. Если выбрать доставку сдек, то появляется калькулятор доставки СДЕК. 
        После того как пользователь укажет адрес доставки мы должны получить эти данные и добавить к заказу. Также к сумме заказа должна добавиться стоимость доставки.

        Ниже есть кнопка Оплатить. Пока что она ведет только на страницу Спасибо за заказ(Просто заглушка с надписью "Спасибо за заказ" и кнопкой вернуться на сайт.)
        Когда пользователь жмет кнопку Оплатить заказ записывается в базу данных, с параметрами Каждого пункта и кол-ва грамм и адреса доставки, а также детализацией цены.
            



        Также при оформлении заказа запрашивается номер телефона и имя, а также пароль. Если номера нет в базе, то создается новый пользователь. Также в хидере справа есть круглая кнопка личного кабинета. Если клиент не залогинен, то на кнопке написано Вход.
        По нажатию появляется окно входа. Если клиент залогинен то по нажатию на кнопку личного кабинета его перебрасывает в личный кабинет. Там есть список всех заказов. Также у клиента хранятся выбранные адреса доставки и по умолчанию, если он уже делал заказы то в калькуляторе доставки сдек устанавливается прошлый адресс.



        Структура базы данных
        Есть коллекции users, products, orders
        users
            _id: генерируемый id
            name: имя
            phone: номер телефона
            password: пароль
            delivery: {
                last: последний пункт доставки
                history:[
                    {
                        'date': дата заказа,
                        "order": id заказа
                    }
                ]
            }
            total: общая сумма трат

            products:
            _id: generated
            name: Black tea
            content: ["названия файлов фото или видео"]
            description: описание чая
            price: 500
            cost: 350
            remains: сколько грамм чая осталось

            orders:
            _id: generated
            date: дата оформления
            list:
                [
                {pid: id товара, count: обьем заказа}
                ]
            delivery:{
                address: адрес доставки
                price: цена доставки
                did: id заказа из системы СДЕК
            }
